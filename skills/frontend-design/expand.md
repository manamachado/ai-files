For each skill:

1. Create a dedicated folder in the Codex skills directory.
2. Add a SKILL.md file.
3. Use YAML front matter.

Format each skill like this:

---
name: <skill-name>
description: Use this skill when performing <specific development task>.
---

# Goal
Explain what the skill is designed to accomplish.

# Instructions
Provide clear step-by-step instructions for Codex to follow when executing the skill.

# Input
Describe the user request or input needed.

# Output
Specify what Codex should generate (code, components, layout, API, etc).

# Best Practices
Include modern development standards, performance considerations, and security guidelines.

Ensure that each skill has a focused responsibility so Codex can automatically select the appropriate skill when handling development requests.
